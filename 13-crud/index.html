Lesson 13 — CRUD

This lesson demonstrates the full Create / Read / Update / Delete lifecycle against a tasks API. The component depends on a TasksService facade — it doesn't know whether the backend is memory or REST. This closes the loop with lesson 08 (Dependency Injection): same pattern, applied to the full data lifecycle.

Swappable backend — the DI payoff

main.js wires new TasksService(new MemoryBackend()) and assigns it as taskManager.service. To switch to a REST backend, replace MemoryBackend with a RestBackend — the component doesn't change. See README.md for the optional json-server setup.
