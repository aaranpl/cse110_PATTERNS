Lesson 07 — Strategy

Strategy is swappable behavior with a uniform interface. All four comparator functions in strategies.js share the same signature: (a, b) => number. The <sortable-list> component knows how to sort — it calls [...this.items].sort(strategies[this.strategy]) — but it does not know how to compare. That's the strategy's job. Swapping the strategy attribute at runtime is all it takes to change the sort order; no logic inside the component changes at all. The uniform signature is what makes the strategies interchangeable.

Looking back: Lesson 04 (Components) showed how to split UI into encapsulated, reusable pieces — <sortable-list> is one such piece. Looking forward: Lesson 08 (Dependency Injection) extends this idea — instead of looking up a strategy by name from a module it imports directly, a component can receive its dependencies from the outside, making it even more decoupled and easier to test.
