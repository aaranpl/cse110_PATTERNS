Lesson 09 — Sortable + Storable List Capstone

This capstone combines lesson 07 (Strategy) for swappable sort behavior with lesson 08 (Dependency Injection) for swappable storage backends. A single <sortable-storable-list> component takes both a strategy attribute (sort comparator) and a storage attribute (backend type) — and either can change independently at runtime without losing the current list of items.

Storage swap behavior: When you change storage, the current items are saved to the newly-selected backend immediately. Switching memory → localStorage persists what you have so far; switching localStorage → memory keeps the items in RAM (and they vanish on page reload). If the new backend already has saved data and the current list is empty, the existing data is loaded instead — so a prior localStorage session survives a page reload automatically.

The two source modules (strategies.js from lesson 07 and storage.js from lesson 08) are duplicated here rather than imported from their original locations, following the curriculum's duplicate-per-lesson rule so each lesson stands alone.
