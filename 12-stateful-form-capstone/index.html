Lesson 12 · Capstone

Stateful Form

This capstone applies the state machine pattern from lesson 10 and the robustness (defensive UI) principles from lesson 11 to a real user interaction: a form submission flow.

The form moves through idle → submitting → success | error states. The state machine from lesson 10 enforces transitions — invalid events are silently ignored rather than causing unpredictable behavior. The defensive UI from lesson 11 drives the user-facing concerns: the input and submit button are disabled while submitting; the submit itself is wrapped in try/catch around an await, and the resulting err.message is shown to the user rather than a generic "something went wrong"; and the error state offers an explicit Retry or Cancel choice rather than leaving the user stuck. A persistent aria-live region announces each state change to screen readers.

The mock submit has a 50% failure rate and a 1.5-second delay — try it a few times to see both paths, and watch the error message vary across the simulated failure modes (503, 500, connection refused).
