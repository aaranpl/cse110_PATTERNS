Notifications Hub Capstone

This capstone combines pub/sub (lesson 01) and delegated routing (lesson 02). Each <status-publisher> child fires a notify CustomEvent that bubbles up through the light DOM to the <notifications-hub> parent, which holds exactly one listener on itself — it routes each event into a per-source stack using event.detail.source. We set composed: true on the event so it can cross shadow DOM boundaries if it ever fires from inside a shadow root; in this demo the event fires on the host element itself, so it already travels through the light DOM. Note: adding a fourth publisher requires no changes to the hub — that's the power of routing by event.detail.source rather than hard-coding handlers per child.
